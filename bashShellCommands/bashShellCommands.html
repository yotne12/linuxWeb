Tutorial using GAS linux

  • Build assembly program with .s extension for 64 bit windows using gcc gas
# ----------------------------------------------------------------------------------------
# Writes "Hello, World" to the console using only system calls. Runs on 64-bit Linux only.
# To assemble and run:
#
#     gcc -c hello.s && ld hello.o && ./a.out
#
# or
#
#     gcc -nostdlib hello.s && ./a.out
# ----------------------------------------------------------------------------------------

        .global _start

        .text
_start:
        # write(1, message, 13)
        mov     $1, %rax                # system call 1 is write
        mov     $1, %rdi                # file handle 1 is stdout
        mov     $message, %rsi          # address of string to output
        mov     $13, %rdx               # number of bytes
        syscall                         # invoke operating system to do the write

        # exit(0)
        mov     $60, %rax               # system call 60 is exit
        xor     %rdi, %rdi              # we want return code 0
        syscall                         # invoke operating system to exit
message:
        .ascii  "Hello, world\n"
  • This line compiles to object code and then runs it.
  • gcc -c hello.s && ld hello.o && ./a.out

A electro-mechanical device is one that uses electricity to physically actuate/move contacts. A motero drives a rotor. On each rotor are contact points at different rotational positions. The rotor then connects different electrical contact points on the stator. A electro-mechanical computer is extremely bulky due to the use of rotors, plug boards and motors. The use of motors (to cchange state by changing the rotational state of rotors) is also very slow compared to other mechanixms. However, give the right circumstances, a elector-mechanical computer can be worthwile. At the endo WW@, German Enigma encryption machines received a fouth wheel to scramble broadcast communication. This made the previous manual deciphering method no loger effective. The bombe computer was devised to semi-automate the deciphering process. Despite being on of the slowest mechanisms, electro-mechanical bombe proved to be effective against the Enigma encryption machines.
